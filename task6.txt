Когда пользователь ввёл в адресной строке браузера имя домена и запустил поиск браузер обращается к резолверу, чтобы найти информацию об IP адресе для домена. 

Резолвер в свою очередь проверяет информацию в локальном кэше, затем обращается к DNS серверам, которые указаны в настройках сети в операционной системе (либо, если настроек нет - к DNS серверу интернет провайдера).
В дальнейшем я буду называть эти сервера кэширующими. 

Кэширующий DNS сервер выполняет поиск по своей базе и, если находит запись об IP адресе для запрашиваемого домена - козвращает резолверу ответ, то в свою очередь передаёт его браузеру. 

Если кэширующий сервер не находит IP адрес в своей базе он начинает рекурсивный DNS запрос. 
В первую очередь запрос уходит к корневым DNS серверам. Таких серверов в мире всего 13 и все они хранят одну информацию - о DNS серверах доменных зон (.ru, .com и т.д.).

В нашем случае корневой сервер возвращает кэширующему информацию о IP адресе DNS серверов зоны .ru

Далее выполняется запрос к DNS серверу зоны, результатом которого будет информация о DNS сервере для нашего домена. 

Кэширующий сервер выполняет финальный запрос к DNS серверу, который используется для домена (авторитативному), получает от него ответ об IP адресе, который указан в записи "А" для домена. 

Затем кэширующий сервер (вероятно), сохраняет эту информацию в кэше для дальнейшего использования и возвращает ответ компьютеру пользователя. 

Таким образом браузер получает от резолвера ответ об IP адресе севрера, на котором расположен сайт. 
Браузер устанавливает соединение с WEB сервером и отправляет HTTP запрос методом GET.

Сервер принимает запрос, формирует ответ. 
Процесс формирования ответа зависит от сайта. 
Сайт может не иметь сложной логики и состоять просто из статичных файлов -  тогда web сервер отвечает на запрос и отправляет эти файлы. 

Также сайт может быть реализован с использованием любого языка программирования (к примеру для взаимодействия с базой данных). 
В этом случае web сервер поступает согласно своим настройкам и обращается к интерпретатору языка, на котором написан сайт. Результат интерпретации кода - файлы, понятные браузеру клиента (html, css, js и т.д.), интерпретатор передаёт их web серверу, а сервер - клиенту. 

После того, как браузер получил ответ от web сервера начинается отрисовка полученного контента в окне браузера.
